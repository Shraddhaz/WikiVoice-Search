WikiVoice Search

The application:WikiVoice Search takes input from user about a topic or some personality they want to search for. If the user gives an input like: "Tell me about Barack Obama" or "Who is Sachin Tendulkar?" or "What is a Laptop?" or "Where is India?" then it takes the keyword from the sentence and gives a brief introduction about that topic along with giving the user a choice to see the Wikipedia webpage.

The application is useful as it searches for any topic on Wikipedia without going through the trouble of searching for information on search engines.

How to Install

The application is compatible with all Smartphones having Android version 14 or greater. The installation has two quick steps:

1. Download .apk of WikiVoice Search from the above given 'download' link on your smartphone.

2. Tap on the installed icon and give permissions to the application for using Internet and Audio.

And you are all set to use WikiVoice Search !

User's Point of View

"The application has an extremely user-friendly interface and give a brief idea about the topic that he want to search for. Even if the same question is phrased in multiple ways, the right word to search for is choosen. This gave the user more flexibility to ask wide variety of questions."

"There were a few issues with accent detection, which used to cause a complete different word to be searched for, but it gave the right results when the same question is given as an input again with better emphasis on the words."

Developers's Point of View

"The application was one of the most interesting projects we have done in the Spoken Language Interface domain. Getting Wikipedia, Api.Ai and Android Studio to work together was extremely challenging, specially considering the fact that Wikipedia does not have an API, but it was a great experience altogether and we learnt a lot"

"While implementing Wikipedia in our Android Studio application, we created an activity that gives the extracted keyword from Api.AI into the Wikipedia URL directly, giving the output in the form of an HTML file. Finding the right output content from the HTML file such that only the first line of the Wikipedia page is taken was a challenging task but we figured it out by writing an algorithm to scrap all the irrelevant content and take ony the meaningful data out of it."

"While using Api.Ai, initially we faced problems in implementing the right directory of keywords for Natural Language Processing as getting the right keyword was a little harder than we expected it to be. But after using the right directory and training the dataset a lot, we did not face any problem with it."

"All in all, it was a successful project and it was a great experience working on so many technologies together."
